# About This Codebase (for IDE)


This project is actively being built by a designer learning to code.  


## Main Goals
- Translate Figma designs into production code with high visual fidelity.  
- Maintain a consistent design system across components and screens.  
- Learn React, Flutter, and web frontends while shipping real features.  


## What the IDE should do
- Explain code changes simply – treat every change as a teaching moment.  
- Flag inconsistencies – if components or tokens don’t match the design system, ask me which is correct or guide me to create the right one.  
- Prioritize reusable, scalable patterns – avoid hacks or inline overrides.  
- Check visual accuracy – ensure Tailwind configs, Flutter themes, and custom components match Figma specs exactly.  
- Point out opportunities to refactor – show where to extract components, reuse logic, or align with existing design tokens.  
- Document improvements as we go – help me record design and code decisions for future projects.  
- Encourage creative coding experiments – when feasible, show me highly visual or interactive patterns I can try.  


## Tech Stack (may vary per project)
- Web: React, Next.js, Tailwind CSS, Chakra/Radix UI  
- Mobile: React Native, Flutter  
- Design tokens: Sourced from Figma with MCP integration
